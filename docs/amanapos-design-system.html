AmanaPOS Design System

The single source of truth for building the AmanaPOS admin dashboard. Every token, component, and pattern — documented and ready to use.

Overview

Introduction

AmanaPOS uses a layered design system built on Tailwind CSS + Radix UI primitives. Two tiers: UI primitives (components/ui/) and DS components (components/ds/).

ℹ️ Rule: Always reach for a component before writing raw Tailwind. If a component doesn't exist for your use case, add it to components/ds/ and document it here.
components/ui/
Radix UI primitives — atoms
Button, Badge, Input, Card, Table, Dialog, Sheet, DropdownMenu, AlertDialog, Avatar, Label, Skeleton, Separator
components/ds/
Dashboard-level molecules
PageTitle, StatCard, Drawer, Pagination, SearchInput, ConfirmDialog, EmptyState, Skeleton variants
Foundation

Design Tokens

All tokens are CSS custom properties defined in globals.css and extended in tailwind.config.ts. Always use the semantic token, never a raw hex value.

Patterns

Page Layout

Every dashboard page follows the same structure. Server component at the root, client components only where interactivity is needed.

// app/[locale]/(dashboard)/owners/page.tsx — SERVER component
export default async function OwnersPage({ searchParams }) {
  const params = await searchParams;
  const page   = Math.max(1, Number(params.page) || 1);

  return (
    <div className="space-y-5">
      <PageTitle
        title={t('title')}
        action={<Button>New Owner</Button>}
      />

      {/* Stats — optional */}
      <Suspense fallback={<StatsSectionSkeleton />}>
        <StatsSection />
      </Suspense>

      {/* Filters — client component */}
      <Suspense fallback={<FiltersBarSkeleton />}>
        <SearchInput placeholder="Search…" />
      </Suspense>

      {/* Table */}
      <div className="bg-card rounded-xl border border-border shadow-card overflow-hidden">
        <Suspense fallback={<TableSkeleton rows={8} cols={5} />}>
          <DataTable searchParams={params} />
          <Pagination count={count} pageSize={20} />
        </Suspense>
      </div>
    </div>
  );
}

Patterns

Forms

All mutations use React 19 useActionState. Never use HTML <form action=…> directly — always wire through a server action.

'use client';

import { useActionState } from 'react';
import { createOwnerAction } from '@/actions/owners';
import { Drawer } from '@/components/ds/Drawer';
import { Input } from '@/components/ui/input';
import { Button } from '@/components/ui/button';

export default function CreateOwnerDrawer({ open, onClose }) {
  const [state, action, isPending] = useActionState(createOwnerAction, null);

  return (
    <Drawer open={open} onClose={onClose} title="New Owner">
      {state?.error && <p className="text-destructive text-sm">{state.error}</p>}
      <Input name="full_name" label="Full name" required />
      <PhoneInput label="Phone" phoneName="phone_local" />
      <Button formAction={action} disabled={isPending}>
        {isPending ? 'Creating…' : 'Create Owner'}
      </Button>
    </Drawer>
  );
}

Patterns

RTL / i18n

Arabic (ar) locale sets dir="rtl" on <html>. All layout must use RTL-safe utilities.

❌ Don't use
ml-4 mr-4 pl-3 pr-3 text-left text-right left-0 right-0 float-left float-right
✓ Use instead
ms-4 me-4 ps-3 pe-3 text-start text-end start-0 end-0 — (use flex instead)
🌐 All user-facing strings must go through useTranslations() in client components or getTranslations() in server components. Add new keys to both messages/en.json and messages/ar.json before committing.
